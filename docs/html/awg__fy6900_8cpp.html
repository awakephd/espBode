The translation table needed for FY6900 AWGs with recent (>= 1.4) firmware.

The table is based on the param_translator structure defined in awg_fy.h. The index of the row matches the id of the parameter (see scpi::param_id). Each entry indicates the type, exponent, precision, and width of values to send, and the type and exponenet of values received.

Recent firmware versions of the FY6900 AWG mostly use floating point (double) values to set frequency, offset, phase, and amplitude. When reading values, frequency is returned as a floating point number, but ammplitude, offset, and phase are returned as integers multiplied by 10^4 (amplitude) or 10^3 (offset and phase). On/Off are represented by integers, where 0 = off and non-zero = on; wave type is represented by an integer representing the id of the wave type (see fy::wave_types).
