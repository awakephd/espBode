◆ ports

enum rpc::ports

Port numbers used in the RPC and VXI communication.

Bind requests always come in on port 111, via either UDP or TCP. Some Siglent oscilloscopes require a different port per link; therefore espBode cycles through a block of ports, changing each time VXI_Server begins listening for a new link request.

Enumerator
BIND_PORT 

Port to listen on for bind requests.

VXI_PORT_START 

Start of a block of ports to use for VXI transactions.

VXI_PORT_END 

End of a block of ports to use for VXI transactions.

◆ reply_state

enum rpc::reply_state

Indicates whether the request was accepted or denied.

Note that messages should be denied only for mismatch of RPC protocol or problems with authorization - neither of which are used in espBode. If accepted, messages will generate a response status which can indicate other types of errors.

Enumerator
MSG_ACCEPTED 

Message has been accepted; its status is indicated in the rpc_status field.

MSG_DENIED 

Message has been denied.

enum rpc::rpc_status
